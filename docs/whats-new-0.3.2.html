💡
From beta (v0.3.2b1): The RAG engine, knowledge graph, and ag index command shipped in the beta. This release adds Smart Model Selection, the TDD mandate in all agent files, ag model, and the full test suite — none of which were in the beta.
🚀

What Changed

🧠 Enterprise Code Intelligence In Beta New

A full knowledge graph and TF-IDF RAG engine is built from your codebase. Agents get injected with real anti-patterns found in your code, and you can search semantically with ag rag "query".

🎯 Purpose-Based Model Routing New

Agentra now uses capability-class routing (inspired by routesmith) to pick the right model for every task type — not just the platform default. Each purpose (planning, coding, review, testing, documentation…) maps to an abstract capability class (deep_reasoning, coding, balanced, fast), which resolves to the best available model on that host. Override any purpose with ag model set <agent> <model> --purpose <purpose>. Every generated agent file shows the full 9-purpose routing table.

🤖 Smart Model Selection New

Agentra now auto-selects the best model per agent platform on ag init. Override any model with ag model set <agent> <model> and list current choices with ag model list. In enterprise environments with model restrictions, use --interactive to pick from a menu or --auto-fallback to automatically select the next available model. Use ag model detect to identify which model is currently active from your environment. Every generated instruction file shows the active model and alternatives.

🔴 TDD Mandate in All Agent Files Improved

The Testing section in every generated file (CLAUDE.md, copilot-instructions.md, .cursorrules, etc.) now opens with an explicit, non-negotiable TDD mandate — Red → Green → Refactor — before the workflow bullets.

🔍 RAG Usage Instructions in Agent Files New

When the knowledge graph is enabled, a new Agentra Code Intelligence block is injected into every agent file telling the AI when and how to run ag rag, ag patterns, and ag index.

🛡️ Machine-Readable JSON Output Fixed Fix

All --format json branches (ag scan, ag index, ag patterns, ag rag) now use plain print() instead of Rich's console — no more ANSI escape codes corrupting piped JSON output.

📦
Shipped in v0.3.2b1 beta. These features are available now on TestPyPI and will be part of the v0.3.2 stable release.

TDD Mandate in Agent Instructions

⚠️
The previous testing section told agents what to do but not when. Agents could still write implementation first, then retrofit tests. The new mandate closes that gap — it is explicit and non-negotiable.
Testing section in all generated agent files Improved
Before (v0.3.1)
## Testing Requirements

### Mandatory Testing Workflow
- **Always write tests** for any new
  or modified code before considering
  a task complete.
- **Run the full relevant test suite**
  after every code change to catch
  regressions immediately.
- Follow the Red-Green-Refactor cycle:
  write a failing test first, make it
  pass, then clean up.
  ...
After (v0.3.2)
## Testing Requirements

### TDD Mandate — Always Follow
### Test-Driven Development
**TDD is non-negotiable.** Every
feature, fix, and refactor follows:
1. **Red** — Write a failing test that
   defines the expected behaviour.
2. **Green** — Write the minimum code
   to make it pass. Nothing more.
3. **Refactor** — Clean up without
   breaking the test.

Never write implementation code before
a test exists for it.

### Mandatory Testing Workflow
  ... (existing bullets preserved)
How this changes agent behaviour in practice
Agent behaviour — Before
# Agent writes feature, then tests:
def paginate(items, size):
    return [items[i:i+size]
            for i in range(0,len(items),size)]

# ... later, adds test:
def test_paginate():  # afterthought
    assert paginate([1,2,3], 2) == ...
Agent behaviour — After
# Agent writes test FIRST (Red):
def test_paginate_splits_evenly():
    assert paginate([1,2,3,4], 2) \
        == [[1,2],[3,4]]

def test_paginate_last_chunk():
    assert paginate([1,2,3], 2) \
        == [[1,2],[3]]

# Then implements (Green):
def paginate(items, size):
    return [items[i:i+size]
            for i in range(0,len(items),size)]

Upgrade Steps

Step 1 — Upgrade the package

pip install --upgrade agentra
# or, from beta:
pip install --upgrade --pre agentra

Step 2 — Re-run ag init to get new blocks in agent files

Running ag init regenerates all agent instruction files with the new Model Preference block, TDD Mandate, and (if indexed) the RAG Usage block. Your existing .agentra.yml config is preserved and extended with model_preferences.

ag init

Step 3 (Optional) — Build your knowledge graph

Required for ag rag, ag patterns, and the codebase anti-pattern block in agent files. Requires scikit-learn (enterprise extra).

pip install "agentra[enterprise]"
ag index
ag init   # regenerate files with RAG block

Step 4 (Optional) — Customise models per agent

ag model list                             # see current selections
ag model set claude claude-opus-4-7       # switch to deep-reasoning model
ag model set copilot gemini-3.1-pro       # switch Copilot model
ag model set cursor gpt-5.3-codex --purpose coding  # override one purpose
🔄

Breaking Changes

None — fully backwards compatible

All new fields in .agentra.yml (model_preferences, rag_config) default to safe values. Old configs are loaded without error. No existing CLI flags changed.
